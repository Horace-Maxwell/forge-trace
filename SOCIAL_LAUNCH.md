## HN body

I built Forge Trace because coding agents create a lot of activity, but not much useful auditability.

Most agent telemetry products are model-first. Forge Trace is software-workflow-first:
- command timeline
- touched files and git diffs
- test runs
- imported PR and review context
- shareable session summary

The goal is to make coding-agent runs easier to inspect, explain, and share.

Repo: https://github.com/Horace-Maxwell/forge-trace
